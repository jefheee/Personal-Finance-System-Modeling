#  Sistema de Finanças Pessoais - Modelagem e Database

Este projeto apresenta a modelagem técnica completa de uma solução robusta para automação e centralização do controle financeiro pessoal, substituindo métodos manuais ineficientes por um ecossistema estruturado em banco de dados relacional.

##  Arquitetura do Projeto
A concepção deste sistema foi baseada em uma análise profunda de regras de negócio, transformando problemas de rastreabilidade em requisitos técnicos. A modelagem foi dividida em três pilares fundamentais:
* **Engenharia de Requisitos:** Identificação de Requisitos Funcionais (RF), Não Funcionais (RNF) e Regras de Negócio (RN) para garantir que o sistema atenda às necessidades reais do usuário.
* **Modelagem UML:** Elaboração de diagramas de Casos de Uso, Classes, Atividades e Sequência para prever o comportamento dinâmico do software.
* **Arquitetura de Dados:** Projeto de um Diagrama Entidade-Relacionamento (DER) otimizado para o SGBD **PostgreSQL**, garantindo integridade referencial e normalização.

##  Diferenciais Técnicos
* **Fluxo de ETL:** Planejamento de um script de Extração, Transformação e Carga (UPDATE via SQL) para automação da migração de dados legados, resolvendo problemas de inconsistência histórica.
* **Dashboard Financeiro:** Prototipagem de uma interface de tomada de decisão com gráficos de pizza e barras para visualização imediata de saldos e despesas por categoria.
* **Usabilidade:** Formulários desenhados para registro rápido de lançamentos, focando na experiência do usuário final.

##  Tecnologias e Ferramentas
* **Modelagem:** UML 2.0 (Casos de Uso, Classe, Sequência).
* **Banco de Dados:** PostgreSQL (DER e Scripts SQL).
* **Design de Interface:** Prototipagem de baixa e alta fidelidade.
